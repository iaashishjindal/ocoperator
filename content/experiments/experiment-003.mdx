## The Problem

I’ve been writing these "experiments" in a chat interface, downloading the markdown, and then manually uploading it to the CMS. It’s tedious, it makes me feel like a digital clerk, and it’s the exact kind of "human-in-the-loop" grunt work that AI is supposed to eliminate. I want to tell Naru "Post this," and have it actually appear on ocoperator.com without me touching a single button.

## The Plan

The goal was simple: 
1. Get an API key for the blog.
2. Teach Naru the schema for our MDX files.
3. Write a small bridge script that takes Naru's output and shoves it into the database via a POST request.
4. Run a test to see if the "plumbing" holds water.

## What Went Wrong (Human Side)

I spent twenty minutes wondering why the API kept returning a `401 Unauthorized` before I realized I had copied the "Secret Key" but missed the last two characters because of a weird line-wrap in the dashboard. 

Also, I totally forgot that our blog engine expects a very specific date format (YYYY-MM-DD), and I just told Naru to "use today's date." It used "March 28th, 2026," which predictably broke the build. I had to go back and be the "adult in the room" and specify the ISO format.

## The AI's Take

Naru is surprisingly confident about API documentation it hasn't seen in six months. It kept trying to use a `/v2/posts/` endpoint that was deprecated back when I still had a regular job. 

When it finally got the endpoint right, it tried to be "helpful" by escaping all the characters in the metadata block, which turned the entire header into a giant, unreadable string of backslashes. It doesn't really "understand" MDX; it just thinks it's looking at a weirdly formatted letter.

## How We Fixed It

*   **The Key Issue:** Re-copied the API key and actually looked at it this time.
*   **The Date Bug:** Added a system instruction that says "You are a robot, use ISO 8601 or die."
*   **The Metadata Mess:** Wrote a specific regex wrapper for the Python bridge to ensure the metadata block remains "raw" while the body gets sanitized.
*   **The Endpoint:** Manually fed the latest API documentation into the context window instead of letting Naru guess.

## The Outcome

If you are reading this on the live site, it worked. This post was generated by Naru, sent through the pipeline, and published without me having to open the admin panel. 

It’s not perfect—the formatting in the "Bill" section still looks a bit wonky in the preview—but the pipeline is open. Now I can focus on the actual experiments instead of acting as a glorified file-transfer protocol.
